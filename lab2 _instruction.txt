OBJECTIVE:To implement and verify 5G Core Network functionality using Open5GS and free5GC, and to design and test REST APIs using Spring Boot with Swagger UI. This lab focuses on hands-on execution and verification of outputs.


PART A: OPEN5GS
Students must:
• Start Open5GS core network services
• Verify that core network components are running

SCREENSHOT REQUIREMENT (MINIMUM 2–3):
1. Open5GS core services running (terminal output)
2. Startup logs 
3. Any verification output (service status or logs)

Each screenshot must include a brief explanation:
• Command executed
• Output observed
• Concept demonstrated


PART B: FREE5GC
Students must:
• Build and run free5GC core network
• Access and verify the free5GC WebConsole
• Verify subscriber configuration or core network status

SCREENSHOT REQUIREMENT (MINIMUM 2–3):
1. free5GC core running using ./run.sh with NF registration logs
2. free5GC WebConsole UI in browser
3. Subscriber creation or verification output (UI or terminal)

Each screenshot must include a brief explanation.

PART C: SPRING BOOT WITH SWAGGER UI
Students must:
• Run the Spring Boot application
• Access Swagger UI
• Test REST APIs using Swagger UI

SCREENSHOT REQUIREMENT (MINIMUM 2–3):
1. Spring Boot application running (terminal output)
2. Swagger UI page
3. Successful API execution (GET / POST response)

Each screenshot must include a brief explanation.
