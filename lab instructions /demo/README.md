# Demo & viva pack

Use these in order for a calm, examiner-friendly presentation:

| Order | File | Use when |
| ---: | --- | --- |
| 1 | `demo_flow.md` | What to say, what to run, fallbacks |
| 2 | `checkpoints.md` | Pass/fail proof per lab |
| 3 | `architecture.md` | One-system story + diagrams |
| 4 | `viva_notes.md` | Short technical Q&A |

Automation: from repo root run **`chmod +x run_demo.sh`** then **`./run_demo.sh`** (or `bash run_demo.sh`). See root **`README.md`**.
